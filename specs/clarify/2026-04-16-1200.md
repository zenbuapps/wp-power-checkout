# Clarify Session 2026-04-16 12:00

## Idea

Power Checkout 是一個 WooCommerce 結帳整合外掛，Settings SPA 使用 Vue 3 + Vue Router，目前使用 Hash 模式 (`createWebHashHistory()`)。

需求是：將 Vue Router 從 Hash 模式改為非 Hash 模式，取消 URL 中的 `#` 符號。

已識別兩種方案：
- 方案 A — `createMemoryHistory()`：URL 不變，路由狀態存在記憶體中，重新整理頁面回到預設路由
- 方案 B — `createWebHistory()` + query param：URL 加上 `&route=/payments/slp`，可 bookmark、可分享，但要動 PHP 側

## Q&A

- Q: 採用哪種方案取代 Hash 模式（A: createMemoryHistory / B: createWebHistory + query param）？影響範圍？ → A: 採用方案 A（createMemoryHistory）。用戶確認只有後台 WC Settings 分頁的 Settings SPA 使用 hash router，其他頁面（RefundDialog、InvoiceApp）不使用 router，無 bookmark/分享需求，符合 Memory 模式的適用場景。

## 已確認事項（程式碼驗證）

1. Router 只在 `js/src/index.ts` 的 `if (form)` 區塊中使用（form = `#mainform`，僅存在於 WC Settings 頁面）
2. `MountRefundDialog()` 和 `MountInvoiceApp()` 在 `if (form)` 之外呼叫，不使用 router
3. `js/src/external/` 目錄下無任何 vue-router import
4. 變更範圍僅限 `js/src/router/index.ts` 一個檔案：`createWebHashHistory()` -> `createMemoryHistory()`
5. 無需修改 PHP enqueue 邏輯、無需拆分 bundle、無需修改 vite.config.ts
